## Next.js 란?

Next.js는 **React 라이브러리의 프레임워크**로 SEO(검색엔진 최적화)를 위한 `SSR(서버 사이드 렌더링)`이 가능하며, `SSG(정적 사이트 생성)`, `ISR(증분 정적 재생성)` 같은 기능들을 제공한다.

React는 라이브러리, Next.js는 프레임워크이다.  
라이브러리와 프레임워크의 차이점은 `제어 흐름에 대한 주도권`이 누구에게, 어디에 있는가의 차이이다. 프레임워크는 전체적인 흐름을 쥐고 있고, 개발자는 그 안에서 라이브러리에 대한 흐름을 쥐고 있다.

## SSR vs CSR

Next.js를 사용하는 큰 이유는 바로 SSR이다.  
SSR(Server-Side-Rendering) 서버 사이드 렌더링은 서버쪽에서 렌더링 준비를 끝마친 상태로 클라이언트에게 전달하는 방식이고, CSR(Client-Side-Rendering) 클라이언트 사이드 렌더링은 렌더링이 클라이언트 쪽에서 일어나는 것을 말하며, 서버로부터 HTML과 자바스크립트를 받는 순간 클라이어트쪽에서 렌더링이 시작되는 방식이다.

SSR은 서버에서 렌더링을 미리 하기 때문에 초기 로딩 속도가 빠르고, 검색 엔진 최적화에 유리하다. CSR은 서버로부터 자바스크립트 파일들을 먼저 다운받은 후 렌더링 하기 때문에 초기 로딩 속도는 느릴 수 있지만, 첫 페이지 로딩 시 모든 파일을 받아오기 때문에 페이지를 이동할 때마다 필요한 데이터만 불러서 사용하기에 페이지 전환이 더 빠르다.

SSR 구현이 필요한 큰 이유 중 하나는 검색 엔진 최적화이다.  
검색 엔진 최적화는 검색 결과 페이지에 노출되고, 사용자 유입을 늘리기 위해서는 절대적이다.

## Next.js 시작하기

Next.js 앱을 만들기 위해서 `create-next-app`을 사용한다.  
타입스크립트롤 사용할 경우 `--typescript`를 추가한다.

```bash
npx create-next-app@latest . --typescript
```

이후 옵션들을 설정해준다.

```bash
Would you like to use ESLint with this project?  Y/N
Would you like to use Tailwind CSS with this project? Y/N
Would you like to use the `src/ directory` with this project? Y/N
What import alias would you like configured? `@/*`
```
